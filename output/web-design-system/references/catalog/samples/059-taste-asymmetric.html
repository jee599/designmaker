Everything tokens should be

Five capabilities that replace six tools.

Live Token Sync

Change a color in your token file and watch it propagate to every component in under 200ms. No build step, no page reload. The sync layer diffs token trees and patches only what changed.

framekit sync --watch --target="./src/tokens.css"

Figma Extraction

Pull spacing, colors, and type scales from any Figma file. Outputs W3C-spec design tokens, not proprietary JSON.

Multi-platform Output

CSS custom properties, Tailwind config, Swift asset catalogs, Kotlin compose themes. One token file, four outputs.

Version History

Every token change is tracked with diffs and timestamps. Roll back to any point in the last 90 days with a single command.

Team Permissions

Designers propose token changes. Engineers review and approve. No one accidentally ships a #000001 that was supposed to be #0a0a0b. Audit log included.

Three steps, not thirty

From design file to production code in minutes.

01

Connect your source

Point Framekit at your Figma file or existing token JSON. It reads your design decisions and maps them to the W3C token spec automatically.

02

Configure outputs

Choose your platforms. CSS variables for web, Swift for iOS, Compose for Android. Each output respects platform conventions — rem units on web, pt on iOS.

03

Ship with confidence

Run the CLI in your CI pipeline. Token changes go through PR review like any other code change. Designers see diffs. Engineers see diffs. Nobody guesses.
