What is the difference between L1 and L2 regularization? When would I choose one over the other in a high-dimensional feature space?
N

L1 (Lasso) adds Σ|wi| as penalty, driving many coefficients to exactly zero. This makes it a de-facto feature selector, useful when you suspect most features are irrelevant.

L2 (Ridge) adds Σwi², shrinking all weights uniformly without eliminating any. It handles multicollinearity better and tends to outperform L1 when most features carry signal.

In high-dimensional settings (p >> n), start with L1 or Elastic Net. If prediction accuracy matters more than interpretability and you have correlated features, L2 is the safer default.

Function calling

Define tools as JSON Schema, and the model decides when and how to invoke them. Supports parallel and sequential tool use.

python 
tools = [{
  "name": "get_weather",
  "description": "Get current weather for a location",
  "parameters": {
    "type": "object",
    "properties": {
      "location": {"type": "string"},
      "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
    },
    "required": ["location"]
  }
}]

response = nexus.complete(
  model="nexus-v4",
  messages=[{"role": "user", "content": "What's the weather in Tokyo?"}],
  tools=tools
)
# response.tool_calls[0].function.name == "get_weather"
# response.tool_calls[0].function.arguments == {"location": "Tokyo"}
